# OPP-011 Batch 1 Execution Checklist

## Goal

Run the first 5-case shadow test for cylinder selection and determine whether agent-selected models are engineering-usable and procurement-positive.

## Part 1: Case Selection

- [ ] pick 5 cylinder cases for Batch 1
- [ ] include 2 recent cases with strong engineering records
- [ ] include 2 historical cases with easy procurement traceability
- [ ] include 1 slightly messy but still comparable case
- [ ] exclude cases with unclear functional requirements
- [ ] exclude cases with missing baseline model records

## Part 2: Baseline Preparation

- [ ] write one requirement summary for each case
- [ ] record the key constraints for each case
- [ ] confirm the baseline engineer-selected model
- [ ] mark whether the baseline model is a preferred model
- [ ] collect baseline price with source
- [ ] collect baseline supplier-option count with counting rule kept consistent

## Part 3: Agent Comparison

- [ ] run the agent on the same requirement boundary for each case
- [ ] record the top recommended model
- [ ] record one backup recommendation if available
- [ ] mark whether the agent recommendation is a preferred model
- [ ] collect agent-side price with source
- [ ] collect agent-side supplier-option count with the same counting rule

## Part 4: Engineering Review

- [ ] arrange engineering review before showing procurement comparison
- [ ] record `pass`, `partial`, or `fail` for each case
- [ ] capture one short review note for each case
- [ ] flag cases where the agent recommendation is not engineering-usable

## Part 5: Procurement Comparison

- [ ] compare baseline and agent price for each complete case
- [ ] compare baseline and agent supplier-option count for each complete case
- [ ] write one per-case conclusion: `better`, `same`, `worse`, or `incomplete`
- [ ] note why any case is incomplete

## Part 6: Batch Review

- [ ] count how many of 5 cases are complete
- [ ] count how many of 5 cases pass engineering usability
- [ ] count how many of 5 cases show a procurement-side positive signal
- [ ] identify which fields were hard to collect
- [ ] identify whether price and supplier-option counting rules need adjustment
- [ ] decide whether to expand to 10 to 20 cases

## Part 7: Procurement Discovery

- [ ] map the current standard-part procurement workflow for cylinders
- [ ] identify who owns supplier choice, preferred-part policy, and price comparison
- [ ] identify the actual standard-part procurement负责人 or closest equivalent role
- [ ] confirm whether engineering and procurement share approval or one side dominates
- [ ] schedule one key-user interview focused on standard-part procurement decisions
- [ ] prepare the procurement interview guide and prep pack

## Expected Outputs

- [ ] completed Batch 1 sample sheet
- [ ] one short Batch 1 summary
- [ ] one list of field or process fixes before Batch 2
- [ ] one procurement interview target list
