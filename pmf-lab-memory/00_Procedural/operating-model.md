# Operating Model

## Purpose

This is the procedural layer for repeated opportunity discovery work.

Current program purpose:

- build a repeatable AI + manufacturing validation mechanism
- prove the team can find opportunities, validate them, and produce decision-grade recommendations
- avoid starting with a broad product build

## What belongs here

- Opportunity intake rules
- Scoring rubric
- Experiment design rules
- Review cadence
- Naming conventions
- Promotion and retirement rules

## What does not belong here

- Live project facts
- Raw source notes
- Unverified insights
- Results that still need review

## Default loop

1. Create or update an opportunity card.
2. Add source notes and evidence.
3. Draft a hypothesis.
4. Design the smallest test that can fail.
5. Run the test.
6. Record outcome and decision.
7. Promote stable patterns to Canonical knowledge.

## Current-stage operating constraints

- team size is two people
- budget and sample access bandwidth are limited
- prioritize small, near-money, easy-to-validate scenarios
- run one main project toward closed-loop validation
- keep A-class learning and C-class falsification active in parallel
- establish the mechanism and cadence before expanding scope

## Funnel

1. Capture opportunities into the opportunity pool.
2. Apply first-pass screening and weighted scoring.
3. Place opportunities into a value-difficulty matrix.
4. Select one primary project and a small number of lightweight validations.
5. Run formal validation through research, prototype, pilot, and ROI estimation.
6. Classify results into A, B, or C outcomes.
7. Decide whether to continue, incubate, pause, or stop.
