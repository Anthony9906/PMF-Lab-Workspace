# Scoring Rubric

## Purpose

Use one weighted rubric to decide whether an opportunity should enter validation.

## First-pass screening gates

- the scenario must be clearly defined
- the target customer, workflow step, pain point, and value-chain position must be describable
- a rough ROI path must exist
- validation difficulty must be estimable
- sample access, data conditions, prototype effort, and cycle time must be judgeable

## Weighted criteria

- strategic fit: 25%
- closeness to money: 25%
- validation difficulty: 20%
- replicability: 15%
- asset accumulation value: 15%

## Scoring guidance

### Strategic fit

- does the opportunity fit the AI + manufacturing focus
- does it fit the current team and company capability boundary

### Closeness to money

- can ROI, pilot value, or willingness to pay be tested within 6 months
- is the value close to cost reduction, efficiency gain, or budget ownership

### Validation difficulty

- can the current two-person team run a minimum validation path
- can the team get samples, domain feedback, and a review loop quickly

### Replicability

- can the opportunity expand from one customer or one workflow to multiple similar cases

### Asset accumulation value

- can the work create reusable data, methods, ROI models, or industry knowledge

## Outcome framework

### A-class

- key assumptions validated
- evidence package completed
- commercialization loop not yet proven

### B-class

- real-scene validation completed
- prototype, feedback, and ROI estimate available
- ready for incubation review

### C-class

- high-quality falsification completed
- current investment is not justified
- restart conditions are explicit

## Decision use

- first-pass screening is not formal project approval
- all projects should pass the same stage gates and review steps
- reward both closed-loop validation and high-quality falsification
