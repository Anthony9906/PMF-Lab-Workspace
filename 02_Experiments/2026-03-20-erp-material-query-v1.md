# ERP物料查询 v1

## 目标

定义一个最小可验证的 `ERP物料查询` 能力，用于从 ERP 物料模块中读取并返回明确、可复核的数据结果。

这不是一个完整的 `ERP-Search Skill`，而是后续 Skill 的最小能力定义和验证基线。

## 本阶段只解决什么问题

本阶段只解决 `物料主数据 / 物料列表` 类页面中的只读查询问题，目标是让 Agent 能稳定回答这类问题：

- 当前 ERP 中某类物料有多少条
- 当前某品牌或某供应商下有多少个有效物料
- 当前某个分类下有多少个物料编码
- 当前启用中的气缸类物料有多少个

默认统计口径：

- 按 `物料编码` 计数
- 返回的是 `记录总数`
- 不做复杂去重

## 本阶段明确不做什么

以下内容不纳入 v1：

- 采购订单查询
- 入库、库存、价格、金额汇总
- “过去 3 个月”这类交易时间统计
- 自动创建、修改、审批、导出后回写
- 严格意义上的“规格去重”
- 跨多个模块拼接结果

原因：

- 这些任务的口径更复杂，容易出现“数字拿到了，但定义错了”的问题。
- 物料模块更适合先验证 Agent 的页面导航、筛选和结果解释能力。

## 用户问题的标准输入形式

每个查询都先被归一化为以下结构：

```text
查询对象：物料
统计口径：物料编码数
筛选条件：品类 / 品牌 / 供应商 / 状态 / 组织
时间范围：无
输出形式：总数 + 筛选条件 + 页面路径 + 备注
```

示例：

```text
问题：ERP 中现在有多少种气缸产品？

归一化后：
查询对象：物料
统计口径：气缸类物料记录数，默认按物料编码计数
筛选条件：品类=气缸，状态=默认页面当前状态或启用中
时间范围：无
```

## 最小支持问题清单

### A. 品类统计

- 现在 ERP 中有多少个气缸类物料
- 现在 ERP 中有多少个标准件物料
- 某个小类下有多少个物料编码

### B. 品牌 / 供应商统计

- 亚德客相关物料有多少个
- 某品牌下当前有效物料有多少
- 某供应商供货物料在 ERP 中有多少条

### C. 状态统计

- 当前启用中的气缸物料有多少
- 当前停用的标准件有多少

## 最小输出格式

每次查询至少返回以下字段：

```text
结果：xxx
口径：按物料编码计数 / 当前列表记录数
筛选条件：xxx
来源路径：模块 > 页面 > 关键筛选
校验：xxx
备注：如有歧义则写明
```

示例：

```text
结果：128 条
口径：按物料编码计数，统计当前物料列表记录数
筛选条件：品类=气缸，状态=启用中
来源路径：基础资料 > 物料主数据 > 物料列表
校验：列表分页总数显示 128，且表格字段包含物料编码
备注：未对规格型号做去重
```

## 页面识别要求

Agent 必须先确认自己进入的是 `正确的物料列表页`，再开始统计。

至少满足以下 3 个特征中的 2 个：

- 页面标题或菜单名包含 `物料`、`存货`、`商品`、`料号` 等同义词
- 表格字段包含 `物料编码`
- 页面支持按 `品类 / 名称 / 编码 / 品牌 / 状态` 进行筛选

推荐的同义词映射：

- 物料：`物料`、`料号`、`商品`、`存货`
- 状态：`启用`、`有效`、`状态`
- 品牌/供应商：`品牌`、`厂商`、`供应商`、`供方`

## 页面上必须识别的关键元素

如果 ERP 物料模块要支持自动查询，最少需要识别这些界面元素：

- 菜单入口
- 页面标题
- 查询区
- 品类筛选项
- 品牌或供应商筛选项
- 状态筛选项
- 查询按钮
- 重置按钮
- 表格区
- 总数显示区或分页总数

## 标准查询流程

### 流程 1：进入页面

1. 打开 ERP 首页或主导航。
2. 找到 `物料主数据 / 物料列表 / 存货档案 / 商品资料` 类页面。
3. 确认页面粒度是 `单条物料记录`，而不是分类树汇总。

### 流程 2：确认默认口径

1. 确认表格存在 `物料编码` 字段。
2. 默认将统计口径定义为 `物料编码数`。
3. 如果页面显示的是分组汇总，不继续统计，先切回明细列表。

### 流程 3：施加筛选

1. 先设置最关键的一级筛选，如 `品类`。
2. 再设置辅助筛选，如 `品牌 / 供应商 / 状态`。
3. 每次修改关键筛选后，都要等待页面刷新完成再读结果。

### 流程 4：读取总数

优先级如下：

1. 页面顶部总数卡片
2. 分页组件中的总记录数
3. 表格工具栏中的 `共 X 条`

如果页面没有明确总数，则本次查询判定为 `不可稳定自动化`，不强行给结果。

### 流程 5：做一次校验

至少做一项校验：

- 校验列表字段中确实包含 `物料编码`
- 校验筛选标签仍然保持在页面上
- 校验分页总数和列表状态没有明显冲突

## 统计口径规则

### 默认口径

- `有多少个物料` = 当前筛选条件下的物料编码记录数

### 需要特别提醒用户的场景

- 用户问“多少种产品”时，系统返回的通常只是记录数，不等于去重后的规格数
- 用户问“多少个规格”时，如果页面没有 `规格型号` 去重能力，必须明确说明当前结果不是严格规格数
- 用户问“亚德客气缸有多少种”时，需要区分：
  - 亚德客品牌下的物料记录数
  - 亚德客气缸规格数

v1 仅承诺第一个口径。

## 失败与中止条件

以下情况不应输出肯定数字：

- 无法确认当前页面是物料明细页
- 页面没有稳定总数
- 筛选条件看起来已生效，但表格没有刷新
- 品类与品牌字段含义不清，容易造成口径错误

此时应返回：

```text
当前无法给出可信结果，原因是 xxx。
建议人工确认页面字段或补充一个更明确的筛选口径。
```

## 第一批验证用例

建议先用 5 个查询做人工对照验证：

1. 当前启用中的气缸类物料有多少个
2. 当前标准件类物料有多少个
3. 亚德客品牌相关物料有多少个
4. 当前停用的气缸类物料有多少个
5. 某个指定小类下有多少个物料编码

每个测试都要记录：

- 问题原文
- Agent 使用的页面路径
- 筛选条件
- Agent 返回结果
- 人工复核结果
- 是否一致
- 不一致原因

## 验收标准

只有满足以下条件，才值得继续往 Skill 演进：

1. 连续 5 个物料查询可以完成
2. Agent 与人工复核结果一致
3. Agent 每次都能说明统计口径
4. 查询过程中没有误点击写操作
5. 同一问题重复执行时结果稳定

## 后续扩展顺序

如果 v1 通过，按这个顺序扩展更合理：

1. 物料规格数查询
2. 供应商-物料关系查询
3. 采购订单数量查询
4. 采购金额和时间区间查询

顺序不要反过来。否则会过早进入高歧义口径。

## 与未来 Skill 的关系

未来的 Skill 不应该一开始就覆盖整个 ERP 查询，而应该先只封装这里定义的最小能力：

- 进入物料模块
- 应用筛选
- 读取总数
- 返回可复核答案

也就是说，下一步如果要写 Skill，正确对象应是：

`ERP Material Query`

不是一开始就做成宽泛的 `ERP Search`。
